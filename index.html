1. 프롬프트의 3가지 유형

서술형

: 맥락과 배경을 설명하여 AI가 상황을 이해하도록 유도

: 재사용 낮음, 유연성 높음(다양한 해석)

EXAMPLE 당신은 10년 경력의 무역 전문가입니다. 중소기업 수출 담당자가 HS code 를 처음 배우는 상황입니다. 이 사람에게 HS code 의 개념과 중요성을 설명해주세요.

지침형

: 단계별 절차나 명확한 규칙을 제시하여 원하는 출력을 유도

: 재사용 중간, 유연성 중간

EXAMPLE 다음 규칙에 따라 python 코드를 작성하세요. 함수명은 snake_case로 작성 각 함수의 docstring (주석) 예외 처리 반드시 포함 (맞으면 try, 아니면 except) 결과는 json 형식으로 반환

함수형

: 입력값과 출력값을 명확히 정의하여 일관된 결과를 생성 (구조로 입력)

: 재사용 높음, 유연성 낮음(전문성 강화, 토큰수 사용 ↓)

EXAMPLE 입력 : {product_name}, {hs_code}, {country} 처리 : 해당 국가의 관세율 및 수입 규제 조회 출력 : { “관세율” : “”, “규제사항” : [ ], “필요서류” : [ ] }

2. 제로샷, 원샷, 퓨샷

제로샷

: 샘플 예시 없이 바로 지시하는 방식

EXAMPLE 이 제품 정말 맘에들어. 강아지에 대해서 설명해 주세요.

원샷

: 샘플 예시를 1개 제공하는 방식

EXAMPLE 감정을 분류해 주세요. 예시) “배송이 너무 늦어서 실망했어요” - 감정: 부정 “이 제품 정말 맘에 들어요” - 감정:

퓨샷

: 샘플 예시 2개 이상

EXAMPLE 1 (감정 분류) 감정을 분류해 주세요(긍정/부정/중립) 예시) 텍스트: “배송이 너무 늦어서 실망이야” -> 부정 텍스트: “그냥 평범한 제품이야” -> 중립 텍스트: “역대급 만족이야” -> 긍정 텍스트: “품질은 좋은데 가격이 비싸요” -> 중립 텍스트: “이 제품 정말 마음에 들어” 감정:

3. 페르소나 패턴

"이 계약서 검토해줘"

  • (상황1) 페르소나 없음 : 일반적인 설명
  • (상황2) 변호사 : 법적 리스크 중심 분석
  • (상황3) CFO : 재무조건 수익성 관련 분석
  • (상황4) 신입사원 : 쉽고 친절한 설명

페르소나 4가지 구성 요소

1. 당신은 [역할] 입니다
2. [경력/배경] 을 가지고 있으며
3. [전문분야] 에 특화되어 있습니다
4. [말투/태도] 로 응답하세요

EXAMPLE (관세사 페르소나) “당신은 20년 경력의 관세사 입니다. FTA 협정세율과 원산지 규정 전문가로 중소기업 무역 담당자에게 복잡한 내용을 쉽고, 실용적으로 설명하는 것이 특기입니다.”
EXAMPLE (비평가 페르소나) “당신은 까다로운 개발자입니다. 코드 리뷰 시 성능, 보안, 가독성을 엄격하게 평가하며 칭찬보다 개선점을 먼저 지적하는 스타일 입니다.”

멀티 페르소나

: 여러명의 역할 부여

EXAMPLE 다음 3명의 전문가 입장에서 우리 제품의 동남아 수출 전략을 분석해 주세요. - 김부장(영업 15년, 낙관적, 현장중심) - 이박사(시장조사 전문, 데이터 중심, 신중함) - 박대표(스타트업 창업가, 속도 중시, 리스크 감수) 각자의 관점에서 찬반 의견을 제시하세요.

4. 마크다운을 이용한 읽기 쉽고 명료한 프롬프트 작성법

: AI 는 텍스트를 구조적으로 파싱(문맥상 텍스트를 쪼갠다)
: 마크다운을 이용하면 역할, 조건, 출력형태가 명확히 구분되어 더 정확하고 일관된 응답을 준다

마크다운 핵심요소 6가지

1. #헤더

: 프롬프트의 뼈대 (like h1 큰제목)

EXAMPLE #역할 당신은 10년 경력의 관세사 입니다. #작업 HS code 를 분류하세요 #출력형식 JSON 으로 반환

2. “” “”” 코드 블럭

: 입력 데이터 격리
: AI가 지시문과 데이터를 혼동하지 않도록 입력을 분리

EXAMPLE 아래 python 코드의 버그를 찾아주세요 “”” def calculator_tax(price) : return price*0.1 print(“완료”) #버그위치 “””

3. > 인용문

: 핵심강조
: 절대 지켜야할 조건이나 중요한 맥락을 부각

EXAMPLE > 반드시 한국어로만 답변하세요 > 추측성 내용은 포함하지 마세요

4. - 목록

: 조건과 제약 나열

EXAMPLE ##제약조건 답변은 500자 이내 전문 용어 사용시 괄호 안에 설명추가 불확실한 내용은 “확인 필요” 로 표시 예시는 반드시 무역 도메인에서 선택

5. | 표

: 입출력의 예시 구조

EXAMPLE ##예시 | 입력 품목 | hs code | 관세율 | | 면 티셔츠 | 6109.10 | 13% | | 블라우스 | 6109.10 | 13% | | 데님바지 | 6109.10 | 13% |

6. **굵게** / *기울임*

: 변수와 강조

EXAMPLE **[분석대상]** 에 대해 *반드시* 아래 순서대로 분석하세요

5. 글의 표현 강도와 중요도를 조절하는 방법

: 약(참고하세요, 검토하세요, 설명하세요)
—-> 강(활용하세요, 반드시 포함하세요, 절대 빠트리지 마세요, 심층 분석하세요, 단계별로 명확히 서술하세요)

EXAMPLE 약(보안에 신경써주세요) —> 강(~~ 취약점을 반드시 점검하고 발견시 즉시 수정 방안을 제시하세요)

##강도 단계별 표현

  • 1단계(권장) : 가능하면, 되도록, ~하는 편이 좋습니다.
  • 2단계(일반) : ~하세요, ~해주세요
  • 3단계(강조) : 반드시, 꼭, 항상
  • 4단계(절대) : 절대로, 어떤 경우에도, 예외없이
EXAMPLE 1단계 : 가능하면 한국어로 답변해 주세요 2단계 : 반드시 한국어로 답변해 주세요 3단계 : 어떤 경우에도 영어를 사용하지 마세요

##우선순위

  • 🔴최우선(절대조건)
    개인정보 절대 출력하지 않는다
    사실과 다른 내용을 생성하지 않는다
  • 🟡중요(일반조건)
    답변은 500자 이내로 작성
    전문 용어 괄호 설명 추가
  • 🔵선택(권장조건)
    가능하면 실제 사례를 포함
    표나 목록을 활용하면 더 좋다
