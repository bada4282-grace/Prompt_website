# 프롬프트 엔지니어링 가이드

## 1. 프롬프트의 3가지 유형

프롬프트는 작성 방식에 따라 **서술형**, **지침형**, **함수형** 세 가지로 분류됩니다.

### 서술형

맥락과 배경을 설명하여 AI가 상황을 이해하도록 유도합니다.
재사용성은 낮고, 유연성이 높아 다양한 해석이 가능합니다.

> **ex.**
> ```
> 당신은 10년 경력의 무역 전문가입니다.
> 중소기업 수출 담당자가 HS code를 처음 배우는 상황입니다.
> 이 사람에게 HS code의 개념과 중요성을 설명해주세요.
> ```

### 지침형

단계별 절차나 명확한 규칙을 제시하여 원하는 출력을 유도합니다.
재사용성과 유연성 모두 중간 수준입니다.

> **ex.**
> ```
> 다음 규칙에 따라 python 코드를 작성하세요.
> - 함수명은 snake_case로 작성
> - 각 함수에 docstring(주석) 포함
> - 예외 처리 반드시 포함 (try / except)
> - 결과는 JSON 형식으로 반환
> ```

### 함수형

입력값과 출력값을 명확히 정의하여 일관된 결과를 생성합니다. 구조로 입력하며, 재사용성이 높고 유연성은 낮습니다(전문성 강화, 토큰 수 절감).

> **ex.**
> ```
> 입력 : {product_name}, {hs_code}, {country}
> 처리 : 해당 국가의 관세율 및 수입 규제 조회
> 출력 : {
>     "관세율" : "",
>     "규제사항" : [],
>     "필요서류" : []
> }
> ```

## 2. 제로샷, 원샷, 퓨샷

### 제로샷

샘플 예시 없이 바로 지시하는 방식입니다.

> **ex.**
> ```
> 이 제품 정말 맘에 들어. 강아지에 대해서 설명해 주세요.
> ```

### 원샷

샘플 예시를 1개 제공하는 방식입니다.

> **ex.**
> ```
> 감정을 분류해 주세요.
> 예시)
> "배송이 너무 늦어서 실망했어요" - 감정: 부정
> "이 제품 정말 맘에 들어요" - 감정:
> ```

### 퓨샷

샘플 예시를 2개 이상 제공하는 방식입니다.

> **ex. 감정 분류**
> ```
> 감정을 분류해 주세요 (긍정/부정/중립)
> 예시)
> 텍스트: "배송이 너무 늦어서 실망이야" -> 부정
> 텍스트: "그냥 평범한 제품이야" -> 중립
> 텍스트: "역대급 만족이야" -> 긍정
> 텍스트: "품질은 좋은데 가격이 비싸요" -> 중립
>
> 텍스트: "이 제품 정말 마음에 들어"
> 감정:
> ```

> **ex. HS code 분류**
> ```
> HS code를 분류해 주세요
> 예시)
> 품목: "면으로 만든 남성용 티셔츠" -> 6109.10
> 품목: "폴리에스테르 소재 여성 블라우스" -> 6206.40
> 품목: "아동용 데님 바지" -> 6203.42
>
> 품목: "울 혼방 남성 재킷"
> HS code:
> ```

## 3. 페르소나 패턴

페르소나를 지정하면 같은 질문에도 전혀 다른 관점의 답변이 생성됩니다.

| 페르소나 | "이 계약서 검토해줘" |
|---|---|
| 페르소나 없음 | 일반적인 설명 |
| 변호사 | 법적 리스크 중심 분석 |
| CFO | 재무조건·수익성 관련 분석 |
| 신입사원 | 쉽고 친절한 설명 |

### 페르소나 4가지 구성 요소

- 당신은 `[역할]`입니다
- `[경력/배경]`을 가지고 있으며
- `[전문분야]`에 특화되어 있습니다
- `[말투/태도]`로 응답하세요

> **ex. 관세사 페르소나**
> ```
> 당신은 20년 경력의 관세사입니다.
> FTA 협정세율과 원산지 규정 전문가로,
> 중소기업 무역 담당자에게 복잡한 내용을 쉽고 실용적으로 설명하는 것이 특기입니다.
> ```

> **ex. 비평가 페르소나**
> ```
> 당신은 까다로운 개발자입니다.
> 코드 리뷰 시 성능, 보안, 가독성을 엄격하게 평가하며
> 칭찬보다 개선점을 먼저 지적하는 스타일입니다.
> ```

### 멀티 페르소나

여러 명의 역할을 동시에 부여하는 방식입니다.

> **ex.**
> ```
> 다음 3명의 전문가 입장에서 우리 제품의 동남아 수출 전략을 분석해 주세요.
> - 김부장 (영업 15년, 낙관적, 현장 중심)
> - 이박사 (시장조사 전문, 데이터 중심, 신중함)
> - 박대표 (스타트업 창업가, 속도 중시, 리스크 감수)
> 각자의 관점에서 찬반 의견을 제시하세요.
> ```

## 4. 마크다운을 이용한 읽기 쉽고 명료한 프롬프트 작성법

AI는 텍스트를 구조적으로 파싱합니다. 마크다운을 활용하면 역할, 조건, 출력 형태가 명확히 구분되어 더 정확하고 일관된 응답을 생성합니다.

### 마크다운 핵심 요소 6가지

**① `#` 헤더** — 프롬프트의 뼈대 역할 (h1 큰 제목처럼)

> **ex.**
> ```
> # 역할
> 당신은 10년 경력의 관세사입니다.
> # 작업
> HS code를 분류하세요.
> # 출력형식
> JSON으로 반환
> ```

**② `"""` 코드 블록** — 입력 데이터 격리. AI가 지시문과 데이터를 혼동하지 않도록 분리합니다.

> **ex.**
> ```
> 아래 python 코드의 버그를 찾아주세요.
> """
> def calculator_tax(price):
>     return price * 0.1
>     print("완료")  # 버그 위치
> """
> ```

**③ `&gt;` 인용문** — 절대 지켜야 할 조건이나 중요한 맥락을 부각합니다.

> **ex.**
> ```
> > 반드시 한국어로만 답변하세요.
> > 추측성 내용은 포함하지 마세요.
> ```

**④ `-` 목록** — 조건과 제약을 나열합니다.

> **ex.**
> ```
> ## 제약조건
> - 답변은 500자 이내
> - 전문 용어 사용 시 괄호 안에 설명 추가
> - 불확실한 내용은 "확인 필요"로 표시
> - 예시는 반드시 무역 도메인에서 선택
> ```

**⑤ `|표|`** — 입출력 예시 구조를 명확히 표현합니다.

> **ex.**
> ```
> ## 예시
> | 입력 품목 | HS code | 관세율 |
> |-----------|---------|--------|
> | 면 티셔츠  | 6109.10 | 13%    |
> | 블라우스   | 6206.40 | 13%    |
> | 데님바지   | 6203.42 | 13%    |
> ```

**⑥ `**굵게**` / `*기울임*`** — 변수와 강조 표현에 사용합니다.

> **ex.**
> ```
> **[분석대상]**에 대해 *반드시* 아래 순서대로 분석하세요.
> ```

### 핵심 요소 6가지 종합 예시

> ```
> # 역할
> 당신은 중소기업 수출 컨설턴트입니다.
> FTA 활용과 원산지 증명 전문가입니다.
>
> # 작업
> 아래 수출 품목에 대한 FTA 적용 가능 여부를 분석하세요.
> """
> 품목명 : 스테인리스 주방용품
> 수출국 : 한국
> 수입국 : 베트남
> """
>
> # 분석기준
> - 한-아세안 FTA 협정세율 적용 여부
> - 원산지 결정 기준 충족 여부
> - 필요서류 목록
>
> # 제약조건
> > 확실하지 않은 세율은 반드시 "세관 확인 필요"로 표기
>
> # 출력
> """json
> {
>     "FTA_적용" : true,
>     "협정세율" : "",
>     "일반세율" : "",
>     "필요서류" : []
> }
> """
> ```

> 프롬프트에는 `[역할, 작업, 입력, 제약, 출력]` 내용이 모두 포함되어야 합니다.

## 5. 글의 표현 강도와 중요도를 조절하는 방법

표현의 강도를 조절하면 AI의 응답 충실도가 달라집니다.

**약한 표현 → 강한 표현 예시**

- 약: `참고하세요`, `검토하세요`, `설명하세요`
- 강: `활용하세요`, `반드시 포함하세요`, `절대 빠트리지 마세요`, `심층 분석하세요`, `단계별로 명확히 서술하세요`

> **ex.**
> - 약: `보안에 신경써주세요`
> - 강: `~~ 취약점을 반드시 점검하고 발견 시 즉시 수정 방안을 제시하세요`

### 강도 단계별 표현

| 단계 | 명칭 | 표현 예시 |
|---|---|---|
| 1단계 | 권장 | 가능하면, 되도록, ~하는 편이 좋습니다 |
| 2단계 | 일반 | ~하세요, ~해주세요 |
| 3단계 | 강조 | 반드시, 꼭, 항상 |
| 4단계 | 절대 | 절대로, 어떤 경우에도, 예외 없이 |

> **ex.**
> ```
> 가능하면 한국어로 답변해 주세요.
> 반드시 한국어로 답변해 주세요.
> 어떤 경우에도 영어를 사용하지 마세요.
> ```

### 우선순위 설정

- 🔴 **최우선(절대조건)**: 개인정보 절대 출력 금지 / 사실과 다른 내용 생성 금지
- 🟡 **중요(일반조건)**: 답변 500자 이내 / 전문 용어 괄호 설명 추가
- 🔵 **선택(권장조건)**: 가능하면 실제 사례 포함 / 표나 목록 활용 권장

## 6. 톤(말투)을 지정하면 전혀 다른 응답이 생성된다

### 톤 변환 예시

> ```
> # 입력
> [원본] = "올해 들어 회사는 큰 변화를 맞이했습니다. 이제 새로운 전략과 목표를 설정하고,
> 모든 직원이 함께 나아가려고 합니다."
>
> # 처리
> 다음 각각의 톤에 맞춰 [원본] 글을 수정해 주세요.
> - 유머러스한
> - 감정적인
> - 따뜻한
> - 아이러니한
> - 비판적인
> ```

### 보고서 작성 시 섹션별 톤 지정

> ```
> # 보고서 작성 지시
>
> ## 요약 섹션
> 임원 보고용: 3줄 이내, 숫자 중심, 공식적 톤
>
> ## 본문 섹션
> 실무자용: 단계별 설명, 기술적이고 구체적 톤
>
> ## 결론 섹션
> 전체 독자용: 행동 유도, 긍정적이고 적극적인 톤
> ```

## 7. 대안 접근법 패턴 : 모든 대안과 해결책 찾아보기

- **일반 프롬프트** → "어떻게 하면 좋을까?" → AI가 임의로 하나 선택
- **대안 패턴** → "모든 방법을 나열하고 각각의 장단점을 비교해줘" → 선택권이 사람에게 돌아옴

> **ex. 일반 프롬프트** → "python으로 엑셀 파일을 읽고 싶어" → AI 답변: openpyxl을 사용하세요
>
> **ex. 대안 패턴** → 방법 1. pandas / 방법 2. openpyxl / 방법 3. xlrd / 방법 4. CSV로 변환

### 패턴 유형 1: 단순 나열형

> ```
> # 지시
> 아래 문제를 해결하는 방법을 최소 4가지 제시하세요.
> 각 방법은 아래 형식으로 작성하세요.
>
> ## 방법 {번호}: {이름}
> - 핵심원리: {한 줄}
> - 적합한 상황: {언제 쓰면 좋은가}
> - 단점: {주의할 점}
> - 난이도: {초급/중급/고급}
>
> # 문제
> 수출통관 서류를 자동으로 생성하고 싶습니다.
> ```

### 패턴 유형 2: 축 기반 비교형

> ```
> # 지시
> 아래 4가지 방법을 다음 기준으로 비교 분석하세요.
>
> # 평가 기준
> - 개발 속도 (빠름/보통/느림)
> - 유지보수 난이도 (쉬움/보통/어려움)
> - 비용 (무료/저비용/고비용)
> - 확장성 (높음/보통/낮음)
>
> # 비교 대상: FastAPI, Django, Flask, Node.js, Express
> # 용도: 무역 데이터 처리 REST API 서버
> ```

### 패턴 유형 3: 상황별 분기형

> ```
> # 지시
> 아래 상황별 가장 적합한 데이터베이스를 추천하세요.
> 각 상황은 독립적으로 판단하세요.
>
> 상황 A: 스타트업, 개발자 1명, 빠른 MVP
> 상황 B: 무역 데이터 10년치, 복잡한 통계 쿼리
> 상황 C: 실시간 환율·가격 데이터, 초당 1000건 업데이트
> 상황 D: 문서 형태의 비정형 통관 데이터 저장
>
> 각 상황에 대해:
> - 1순위 추천: {DB명} - {이유 한 줄}
> - 2순위 추천: {DB명} - {이유 한 줄}
> ```

### 패턴 유형 4: 역발상 대안형

> ```
> 아래 [문제]에 대해 일반적인 해결책뿐 아니라
> "하지 않는 선택지"와 "문제 자체를 재정의"하는 방법도 대안으로 포함하여
> 총 5가지를 제시하시오.
>
> [문제] = 수입 통관 처리 시간이 너무 오래 걸린다.
> ```

## 8. 이용자 페르소나 패턴 : 대상에 특화된 맞춤형 답변 유도하기

- **페르소나 패턴** → AI의 역할 정의
- **이용자 페르소나 패턴** → "이 답을 듣는 사람은 ~~~입니다"

> **ex. 동일 질문, 다른 이용자**
> ```
> 질문: "블록체인이 무역금융에 어떻게 활용되나요?"
>
> 이용자1: 무역을 처음 배우는 대학생
> 이용자2: 중소기업 수출 담당 팀장
> 이용자3: 금융기관 IT 의사결정자
> ```

> **ex. 이용자 페르소나 적용 예시**
> ```
> # 이용자 정보
> - 직책: 중소기업 대표 (제조업, 직원 30명)
> - 수출 경험: 국내 영업만 10년, 해외 수출 처음
> - 관심사: 리스크 최소화, 빠른 실행
> - 우려 사항: "복잡하고 어려울 것 같다"는 막연한 두려움
> - 시간: 바빠서 긴 글을 읽지 않음
>

> ```

## 9. 레시피 패턴 : 목표 달성을 위한 작업 단계 받기

구체적인 순서를 요구할 때 사용합니다.

- **레시피 패턴**: "수출 자동화를 위한 단계별 절차를 주세요" → 순서 있는 행동 지침

### 기본 단계형

> python 초보자가 무역 HS code 자동 분류 프로그램 만들기
>
> # 현재 상태

> - 프로그램 경험 없음

>
> ## STEP {번호}: {단계명}
> - 할 일: {구체적 행동}
> - 소요 시간: {예상 기간}
> - 완료 기준: {이것이 되면 다음으로}
> - 주의사항: {흔한 실수}

> ```
> 무역 계약 클레임 처리 절차를 단계별로 작성하세요.
> 단, 각 단계에서 발생할 수 있는 분기 상황도 포함하세요.
>
> 형식:
> STEP N. (행동)
> - 정상: {이 결과면 다음 단계로}

### 체크리스트형

> ```

> [ ] {항목} - {이유 또는 주의사항}

## 13. 질문 개선 패턴 : 질문을 질문하기

AI에게 사용자의 질문 자체를 먼저 분석하고 더 나은 질문으로 업그레이드하도록 지시합니다.

### 방법 1: 개선 후 답변

> ```
> 제 질문에 바로 답하지 말고,
> 먼저 제 질문을 분석해서 더 나은 질문으로 개선해 주세요.
> 개선된 질문을 보여준 뒤, 그 질문에 스스로 답하세요.
> ```

### 방법 2: 여러 버전 제시

> ```
> 제 질문을 3가지 다른 방향으로 개선해 주세요.
> 각 버전이 어떤 목적에 맞는지 설명하고,
> 제가 선택하면 그 버전으로 답하세요.
> ```

### 방법 3: 문제 진단 먼저

> ```
> 제 질문의 문제점 (모호함, 잘못된 전제, 누락된 맥락)을 먼저 지적하고,
> 수정된 질문으로 답하세요.
> ```

### 활용 예시

> **단순 개선형**
> ```
> [질문] = "AI를 무역에 활용하려면 어떻게 해야 하나요?"
>
> 위 [질문]을 더 명확하고 답변 가능한 형태로 개선한 뒤 개선된 질문으로 답변하세요.
> 개선한 이유도 한 줄로 설명하세요.
> ```

> **다방향 분기형**
> ```
> [질문] = "수출 자동화는 어떻게 시작하나요?"
>
> 제 [질문]을 3가지 다른 해석으로 개선해 주세요.
> 각 버전이 어떤 상황에 맞는지 설명하고,
> 제가 고르면 그것으로 답하세요.
> ```

> **전제 교정형**
> ```
> [질문] = "FTA를 쓰면 관세를 안 내도 되는 건가요?"
>
> 제 [질문]에 잘못된 전제나 오해가 있으면 먼저 그것을 지적하고 교정한 뒤,
> 교정된 질문에 답하세요.
> ```

### 질문 보내기 전 체크리스트

- 대상이 명확한가?
- 범위가 좁혀졌는가?
- 목적이 포함되어 있는가?
- 현재 상태가 포함되어 있는가? (예: python 왕초보 / 기초 / 중급)

## 15. 메타언어 생성 패턴 : AI와 대화하는 나만의 비밀 코드

둘만의 약어를 `/슬러시 + 축약어` 형태로 만들어 재사용할 수 있습니다.

### 예시 1: 무역 실무 단축어 세트

> ```
> 이 대화에서 아래의 명령어를 사용합니다.
> 각 명령어의 정의를 정확하게 기억하고 실행하세요.
>
> /요약 = 핵심만 3줄, 전문용어 제거, 무역 실무자 기준, 결론 먼저 → 이유 순서로 작성
> /코드 = python 작성, 주석 한국어, 예외처리 포함, 실행 가능한 완성 코드, 설명 없이 코드만 출력
> /비교 = 2~4개 대안을 표로 정리,
>         항목: 장점, 단점, 비용, 난이도, 추천 상황,
>         마지막 행에는 종합 추천 추가
>
> 사용자: /요약 베트남 FTA 활용 방법 알려줘
> ```

### 예시 2: 어휘 단축어 세트 (편집자 역할)

> ```
> # 입력
> [독자] = 초등학생
> [단축어1] = "/유:[단어]:[개수]" 라고 요청하면,
>             [단어]에 대해 [독자]가 이해할 수 있는 유의어를 [개수]만큼 제공하고,

## 16. 리플렉션 패턴 : AI를 위한 자기성찰의 거울

AI 초안 → AI 스스로 비평 → AI 스스로 개선 작업 → AI 최종 결과물 생성

### 방법 1: 생성 → 비평 → 개선

> ```
> 아래의 작업을 3단계로 수행하세요.
>
> 1단계 [초안]: 요청에 대한 첫 번째 답변 작성
> 2단계 [비평]: 초안의 약점, 누락, 개선점을 스스로 분석
> 3단계 [개선]: 비평을 반영한 최종 버전 작성
> ```

### 방법 2: 반복 정제

> ```
> 초안을 작성하고 N번 스스로 개선하세요.
> 각 반복마다 무엇을 왜 바꿨는지 한 줄로 기록하세요.
> ```

### 3단계 리플렉션 예시

> ```
> 아래 요청을 처리하되 반드시 3단계로 진행하세요.
>
> [초안]
> 먼저 최선을 다해 답변을 작성하세요.
>
> [자기 비평]
> 방금 작성한 답변을 검토하고 아래를 솔직하게 답변하세요.
> - 가장 약한 부분은 어디인가요?
> - 빠뜨린 중요한 내용이 있나요?
> - 독자가 오해할 수 있는 부분이 있나요?
>
> [개선본]
> 비평을 반영한 최종 답변을 작성하세요.
>
> 요청: "python으로 엑셀 파일을 읽어 HS code별로 정렬하는 코드 작성"
> ```

### 자기 평가 후 개선 예시

> ```
> 아래 문서를 작성하고 완성 후 5가지 기준으로 각각 10점 만점 평가를 수행하세요.
> 점수가 7점 미만인 항목은 반드시 개선하세요.
>
> 평가 기준:
> ① 명확성: 한 번에 이해되는가?
> ② 완결성: 필요한 내용이 모두 있는가?
> ③ 정확성: 틀린 내용이 없는가?
> ④ 실용성: 읽은 후 바로 실행할 수 있는가?
> ⑤ 간결성: 불필요한 내용이 없는가?
>
> 요청: "첫 수출을 준비하는 소기업을 위한 체크리스트 작성"
> ```

> - 협상 전략 3단계

> 1. 현재 상황 분석

> # 출력

> 3. 협상 시나리오: 단계별 양보안(BATNA)과 최종 합의 조건을 포함한
>                  3가지 시나리오 제시
>
> Expectations:
> - 법적 대응 시 예상되는 소송 비용 및 기간 산출
> - 비즈니스 관계 유지를 최우선으로 하되, 채권 회수율 85% 이상 달성 논리 제공
> - 감정적 대응을 배제하고 데이터와 국제 관례에 기반한 전문적 톤 유지
>
> Narrowing:

## 25. BAB 프레임워크 : 수출 성공 스토리

상대방을 설득하거나 변화의 가치를 증명해야 하는 마케팅 및 카피라이팅에 적합합니다.
광고 문구 작성, 이메일 마케팅, 서비스 도입 전후 비교를 통한 영업 제안에 활용됩니다.

**구성 요소**

| 요소 | 설명 |
|---|---|
| `Before` | 문제 상황 — 독자가 현재 겪고 있는 고통·불편을 묘사 |
| `After` | 이상적 미래 — 문제가 해결된 뒤의 긍정적 상태를 제시 |
| `Bridge` | 연결 고리 — Before에서 After로 가기 위한 해결책(우리 제품·서비스) 제안 |

> ```
> Before:
> {현재 독자가 겪고 있는 문제 상황}
>
> After:
> {문제가 해결된 이후의 이상적인 상태}
>
> Bridge:
> {Before에서 After로 전환시켜 주는 해결책 또는 제안}
> ```
## 26. 다중 관점 기법 : 여러 관점으로 보기

다양한 관점을 통해 문제를 분석하고, 상호 교차점을 찾아 더 나은 해결책을 도출하는 기법입니다.

### 다중 관점의 종류
- **전문가 관점**: 실용성은 높지만, 감성이 누락될 수 있습니다.
- **사용자 관점**: 공감은 높지만, 전문성이 누락될 수 있습니다.
- **낙관론**: 가능성은 보이지만, 리스크가 누락될 수 있습니다.
- **비관론**: 리스크는 보이지만, 기회가 누락될 수 있습니다.

### 다중 관점이 도출하는 결과물
- 관점이 **충돌**하는 지점 → 진짜 문제 발견
- 관점이 **동의**하는 지점 → 확실한 진실 발견
- 관점이 **보완**하는 지점 → 완전한 해결책 발견

### 다중 관점 5가지 원칙
1. **이해관계자 차원**: 관련된 사람들의 서로 다른 입장 (예: 사장 vs 직원)
2. **전문 분야 차원**: 다른 학문과 직종의 시각
3. **시간 차원**: 과거, 현재, 미래의 시각
4. **가치관 차원**: 다른 철학과 우선순위
5. **문화/맥락 차원**: 다른 환경 및 배경의 시각

### 프롬프트 적용 패턴

> **방법 1: 병렬 관점 소환**
> ```text
> 아래 주제를 {n차원 관점}에서 동시에 분석하세요.
> 각 관점은 독립적으로 분석하되,
> 마지막에 관점들이 만나는 교차점을 찾아주세요.
> ```

> **방법 2: 대화 시뮬레이션**
> ```text
> {관점 A}, {관점 B}, {관점 C}가 같은 테이블에 앉아 이 주제를 토론합니다.
> 각자의 주장, 반론, 그리고 결론을 써 주세요.
> ```

> **방법 3: 레이어 분석**
> ```text
> 같은 주제를 {레이어1}에서 {레이어n}까지 겹겹이 쌓아가며 분석하세요.
> 아래 층위가 위 층위를 어떻게 규정하는지 보여주세요. (아래에서 위를 바라보는 관점)
> ```

### 다중 관점 프롬프트 종합 예시

> **[예시 1] 이해관계자 차원 분석**
> ```text
> 아래 상황에 관련된 4명의 이해관계자를 동시에 분석하세요.
>
> # 각 이해관계자 분석 항목:
> - 가장 원하는 것 (핵심 욕구)
> - 가장 두려워하는 것 (핵심 공포)
> - 공개적으로 말하는 것 vs 속으로 생각하는 것
> - 이 상황에 갖는 협상력
>
> # 최종 도출 항목:
> - 모두가 동의할 수 있는 최소 공통점
> - 가장 큰 갈등 지점
> - 현실적 합의 경로
>
> 상황: 중소기업이 수출 자동화 시스템을 추진 중인 상황
> 이해관계자: 대표, 무역팀장, 팀원 2명, IT 외주업체
> ```

> **[예시 2] 전문 분야 차원 분석**
> ```text
> 아래 문제를 5가지 {전문 분야}의 렌즈로 동시에 분석하세요.
> 각 분야가 이 문제를 어떻게 정의하고 어떤 해결책을 제안하는지 보여주세요.
>
> 분야: 심리학 / 경제학 / 시스템 공학 / 인류학 / 게임 디자인
> 문제: “무역팀이 새로운 자동화 도구 도입 후 3개월이 지나도 제대로 활용하지 못한다”
> ```

> **[예시 3] 시간 레이어 분석**
> ```text
> 아래 주제를 3개의 시간 레이어에서 동시 분석하세요.
> 각 레이어가 다음 레이어에 어떤 영향을 주는지 연결하세요.
>
> - 레이어 1 (과거, 5년 전): 당시 사람들은 무엇을 알고 무엇을 몰랐나?
> - 레이어 2 (현재): 과거의 결정이 현재에 어떤 모습으로 나타나는가? 지금 진짜 문제는 무엇인가?
> - 레이어 3 (미래, 5년 후): 지금의 선택이 미래에 어떤 모습으로 누적되는가? 어떤 결정을 지금 내려야 하는가?
>
> 주제: “한국 중소기업의 무역 디지털 격차”
> ```

## 27. 심사숙고 유도 기법

AI가 확률적으로 가장 가까운 답을 즉시 생성하지 않고, 자기 검증과 추론(Chain Of Thought, COT)을 거친 뒤 답변하도록 통제합니다.

### 5대 통제 메커니즘
- **메커니즘 1 (사고 공간 확보)**: 답을 쓰기 전에 먼저 생각하도록 지시
- **메커니즘 2 (사고 단계 강제)**: 건너뛸 수 없는 중간 단계를 명시적으로 설계
- **메커니즘 3 (자기 검증 루프)**: 답을 낸 후 스스로 검토하는 과정 추가
- **메커니즘 4 (불확실성 명시)**: 아는 것과 모르는 것을 구분하게 하여 환각 방지
- **메커니즘 5 (반론 요구)**: 도출된 결론에 대한 스스로의 반론 작성 지시 (선택적 활용)

### COT(Chain of Thought) 적용 비교

> **수정 전 (Before)**
> ```text
> 우리 회사가 베트남에 진출해야 할까?
> ```
>
> **수정 후 (After)**
> ```text
> 당신은 20년 차 전략 컨설턴트(COT 마스터)입니다.
> 우리 회사가 베트남에 진출해야 할지 '단계별로 생각(Chain of Thought)'하면서 판단해 주세요.
> 반드시 각 단계에서 내린 중간 결론을 먼저 보여준 뒤 최종 결론을 제시하세요.
> ```

## 28. APE (Automatic Prompt Engineer) 프레임워크

AI를 단순한 답변 생성기가 아닌, 최적의 프롬프트를 자동으로 생성, 평가, 개선하는 메타 도구로 활용하는 기법입니다.

### 수동 APE 마스터 프롬프트 예시

> ```text
> # 역할
> 당신은 프롬프트 엔지니어링 전문가입니다.
> 주어진 작업에 대해 최적의 프롬프트를 자동으로 생성, 평가, 개선합니다.
>
> ## APE 실행 순서
>
> ### Step 1: 후보 프롬프트 생성
> 아래 작업을 위한 프롬프트 후보 10개를 생성합니다. 각 후보는 아래의 전략 중 서로 다른 것을 사용해야 합니다.
> - 역할 부여형 / 단계별 지시형 / 예시 포함형 / 제약 명시형 / 목적 중심형 / 독자 설정형 / 형식 지정형 / 품질 기준형 / 역방향형 / 메타 지시형
>
> ### Step 2: 각 후보 자체 평가
> 각 프롬프트에 대해 아래 기준으로 점수(100점 만점)를 매기세요.
> - 명확성(25점): 지시가 모호하지 않은가
> - 완전성(25점): 필요한 정보가 다 있는가
> - 실행 가능성(25점): AI가 바로 실행하는가
> - 품질 유도력(25점): 높은 품질의 결과를 이끌어 내는가
>
> ### Step 3: 교차 최적화
> 상위 3개 프롬프트의 강점을 결합하여 최종 최적 프롬프트 1개를 생성합니다.
>
> ### Step 4: 최적 프롬프트 출력
> - 최적 프롬프트 전문 출력
> - 이 프롬프트가 최선인 이유 설명
> - 사용 시 주의사항 명시
>
> ### 작업 입력:
> 작업: {고객 클레임 이메일 답장 작성}
> 원하는 결과물: {고객이 화가 풀리고 우리 회사를 신뢰하게 하는 답장}
> 피해야 할 것: {방어적 태도, 책임 회피, 공허한 사과}
> ```

### 코딩 작업 전용 APE 프레임워크

**코딩 프롬프트 품질 기준**:
- **실행 가능성 (30점)**: 코드가 수정 없이 즉시 실행되는가? (에러 처리 포함 여부)
- **완전성 (25점)**: 엣지 케이스 및 예외 처리가 반영되었는가?
- **가독성 (25점)**: 주석과 변수명이 직관적이고 이해하기 쉬운가?
- **확장성 (20점)**: 추후 수정 및 다른 케이스 응용이 용이한 구조인가?

> **코딩 APE 프롬프트 템플릿**
> ```text
> 다음 코딩 작업을 위한 최적 프롬프트를 생성하세요.
>
> 작업: {무엇을 만들 것인가}
> 언어/프레임워크: {python / javascript / etc ...}
> 제약: {성능 요구, 라이브러리 제한 등}
> 경험 수준: {초보 / 중급 / 고급}
> ```

### 코딩 프롬프트 고도화 과정 (진화형 예시)
- **Level 1**: `파이썬으로 엑셀 파일 읽는 코드 만들어줘`
- **Level 2 (라이브러리 지정)**: `pandas로 엑셀 파일을 읽는 파이썬 코드를 작성하세요.`
- **Level 3 (제약 조건 추가)**: `pandas로 엑셀 파일을 읽는 코드를 작성하되, 파일이나 시트가 없을 경우의 에러 처리를 반드시 포함하세요.`
- **Level 4 (구조화 및 기능 명시)**: `기능(엑셀 읽기 + 기본 전처리)과 포함 사항(...)을 만족하는 파이썬 코드를 작성하세요.`
- **Level 5 (이용자 페르소나 적용)**: `초보 파이썬 개발자를 위해, 실무에 바로 쓸 수 있고 이해하기 쉬운 엑셀 처리 코드를 상세한 주석과 함께 작성해 줘.`

## *AI 프롬프트 작성법 : 5가지 핵심 열쇠

**1. 명확하고 구체적으로 지시하기**
- 수정 전: `마케팅 좀 알려줘`
- 수정 후: `월 예산 50만 원으로 40대 타겟의 매출을 20% 올릴 아이디어 3개 제안해 줘.`

**2. Context (충분한 배경 정보 제공)**
- 수정 전: `은퇴 준비 글 써줘`
- 수정 후: `나는 10년 차 재무 상담사야. 불안해하는 50대 직장인에게 따뜻한 조언조로 써줘.`

**3. 말보다 샘플(Few-Shot) 하나 보여주기**
- 수정 전: `감동적인 이메일 써줘`
- 수정 후: `내가 쓴 이 예시와 비슷한 톤앤매너로 고객 감사 메일 써줘. [샘플 데이터 첨부]`

**4. 거대한 업무는 잘게 쪼개기**
- 수정 전: `사업계획서 한 번에 짜줘`
- 수정 후: `먼저 핵심 컨셉 3개 제안해 줘. 그중 하나를 골라 상세 목차를 짜보자.`

**5. AI를 파트너로 대하기**
- **개념**: 한 번의 프롬프트로 완벽한 결과물을 기대하기보다 '초안'으로 인식하고 피드백 루프 생성.
- **활용 예시**: "이 부분은 좀 더 전문가답게 고쳐줘", "무역 실무 사례를 하나 더 추가해 줘" 등 반복 정제 수행.
